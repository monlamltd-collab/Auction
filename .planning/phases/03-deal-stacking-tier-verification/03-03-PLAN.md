# Plan 03-03: Tier Verification & Edge Cases

## Objective
Systematically verify all subscription tier lifecycle flows end-to-end and ensure every frontend gating point works correctly. This includes trial expiry, resubscription, payment failure grace periods, and the edge case where a user with an expired trial subscribes via Stripe. Also implement cross-tab tier synchronisation and create a comprehensive verification checklist.

## must_haves
- [ ] Trial expiry correctly downgrades user to free tier (TIER-01)
- [ ] Resubscription after cancellation restores premium access (TIER-02)
- [ ] Payment failure triggers 3-day grace period before downgrade, not immediate cutoff (TIER-03)
- [ ] All frontend gating points verified: AI search limits, lot blur, CSV/JSON export, affordability filters, yield analysis, comparables, deal stacking (TIER-04)
- [ ] User with expired trial subscribes via Stripe and is correctly activated as premium (TIER-05)
- [ ] Cross-tab tier synchronisation works (upgrade in one tab reflected in others)
- [ ] Verification checklist document created covering all backend lifecycle paths and frontend gates

## Tasks

<task id="03-03-01">
<title>Implement cross-tab tier synchronisation via localStorage</title>
<details>
In `index.html`, add a tier sync mechanism using localStorage events:

**On tier state change (in `updateAccountUI()` around line 1798):**
After setting `window._userTier` from the `/api/stripe/status` response, also write to localStorage:
```js
localStorage.setItem('bridgematch_tier', window._userTier);
localStorage.setItem('bridgematch_tier_ts', Date.now().toString());
```

**On page load / auth state change (early in the script, near the auth listener setup):**
Add a `storage` event listener that detects tier changes from other tabs:
```js
window.addEventListener('storage', function(e) {
  if (e.key === 'bridgematch_tier' && e.newValue && e.newValue !== window._userTier) {
    window._userTier = e.newValue;
    // Re-render any visible gated content
    updateAccountUI();
  }
});
```

This ensures that when a user upgrades in tab A, tab B picks up the change instantly without requiring a manual refresh or polling.

**Edge case:** If `updateAccountUI()` itself writes to localStorage, this could cause an infinite loop. Guard against it:
```js
window.addEventListener('storage', function(e) {
  if (e.key === 'bridgematch_tier' && e.newValue && e.newValue !== window._userTier) {
    window._userTier = e.newValue;
    // Only update UI, don't re-fetch status (which would write to localStorage again)
    // Call a UI-only refresh function instead of full updateAccountUI()
    refreshTierUI();
  }
});
```

Create a `refreshTierUI()` function that updates the account button text and any visible gated sections without calling `/api/stripe/status` again. This prevents the infinite loop while keeping the UI current.
</details>

3. Switch to tab B — verify it detected the change and updated `window._userTier` to 'premium'
4. Verify no infinite loop in the console (no rapid-fire storage events)
</verify>
</task>

- Store `stripe_subscription_id`
- Clear `tier_expires_at`

5. Confirm frontend shows premium access

**TIER-03: Payment failure → 3-day grace**

The `customer.subscription.updated` webhook handler should, when status is `past_due`:
- Keep `tier='premium'`
- Set `tier_expires_at` to `now + 3 days`

Verification:
1. Use Stripe CLI or test dashboard to simulate a payment failure on an active subscription
2. Confirm webhook fires with `status: past_due`
3. Confirm user remains `tier='premium'` but `tier_expires_at` is set to 3 days from now
4. Manually set `tier_expires_at` to the past to simulate grace expiry
5. Make an authenticated request — confirm downgrade to free

**TIER-05: Expired trial + subscribe → premium**

This is the edge case where `trial_used=true`, `tier='free'`, and the user completes a Stripe checkout.

Verification:
1. Take a user with `trial_used=true`, `tier='free'`
2. Complete a Stripe test checkout
3. Confirm `checkout.session.completed` handler upgrades them to `tier='premium'`
4. Confirm `tier_expires_at` is cleared (not inherited from old trial)
5. Confirm `trial_used` remains true (no new trial)

**If any issues found:** Fix in the relevant webhook handler in `server.js`. The research indicates these flows should work correctly, but verification may reveal edge cases.
