# Plan 03-01: Deal Stacking Calculator Core

## Objective
Build the deal stacking calculator as a client-side JS feature that replaces the broken `calcDealAnalysis()` function. The calculator takes a lot's purchase price, user-inputted GDV, works cost, and optional rental income, then produces a full investment stack for both Flip (sell) and Hold (refinance + rent) scenarios. It integrates with the existing `calcSDLT()` function and `LENDER_DATA` array, and is controlled by the existing LTV slider in the Finance Check widget.

## must_haves
- [ ] `calcDealAnalysis()` is removed and replaced with `calcDealStack()` implementing correct deal stacking logic
- [ ] User can input GDV, works cost, and optional rental income per lot in the expanded panel
- [ ] SDLT is auto-calculated from lot guide price using `calcSDLT()` with correct country detection
- [ ] Bridging finance costs sourced from best-match lender in `LENDER_DATA`; falls back to market averages (0.85%/mo, 2% arrangement) when unavailable
- [ ] Full investment stack displayed: total cost in, net profit, ROI, cash-on-cash return
- [ ] Flip and Hold scenarios shown side-by-side; Hold requires rental income input
- [ ] LTV slider change triggers deal stacking recalculation alongside Finance Check
- [ ] Results recalculate live as user types (debounced 300ms)
- [ ] Itemised cost breakdown visible (solicitor, survey, broker fee, arrangement fee, bridging interest)

<task id="03-01-01">
<title>Implement calcDealStack() function replacing calcDealAnalysis()</title>
<details>
In `index.html`, locate `calcDealAnalysis()` at approximately line 2539. Remove the entire function and replace it with a new `calcDealStack()` function.

**New function signature:**
```js
function calcDealStack(purchasePrice, gdv, worksCost, rentalIncome, ltvPct, address) {
```

**Lender matching logic** (inside the function):
1. Filter `window.LENDER_DATA || []` to lenders where `_parseLTV(l.l1).pct >= ltvPct`
2. Among eligible lenders, pick the one with the lowest `_parseRate(l.r)` — store as `bestLender`
3. If no match or LENDER_DATA empty, set `usingFallback = true`, use rate = 0.0085 (0.85%/mo), procFee = 0.02 (2%)
4. Otherwise: `monthlyRate = _parseRate(bestLender.r)`, `procFeePct = _parseProcFee(bestLender.pf) / 100`

**Shared cost calculations:**
- `sdlt = calcSDLT(purchasePrice, detectCountry(address))`
- `loanAmount = purchasePrice * (ltvPct / 100)`
- `bridgingInterest = Math.round(loanAmount * monthlyRate * 12)` (12-month bridge term)
- `arrangementFee = Math.round(loanAmount * procFeePct)`
- `solicitor = 1500`
- `survey = 500`
- `brokerFee = Math.round(loanAmount * 0.01)`
- `totalCostIn = purchasePrice + sdlt + worksCost + bridgingInterest + arrangementFee + solicitor + survey + brokerFee`
- `cashIn = (purchasePrice - loanAmount) + sdlt + worksCost + solicitor + survey + brokerFee`

**Flip scenario:**
- `netProfit = gdv - totalCostIn`
- `roi = cashIn > 0 ? (netProfit / cashIn * 100) : 0`
- `cashOnCash = roi` (same as ROI for flip)

**Hold scenario (only if rentalIncome > 0):**
- `refiAmount = Math.round(gdv * 0.75)` (BTL 75% LTV)
- `btlMortgage = Math.round(refiAmount * 0.055 / 12)` (5.5% interest-only annual)
- `management = Math.round(rentalIncome * 0.10)`
- `voidAllowance = Math.round(rentalIncome / 12)` (4-week void as 1/12 of monthly)
- `monthlyCashflow = rentalIncome - btlMortgage - management - voidAllowance`
- `annualYield = purchasePrice > 0 ? ((rentalIncome * 12) / purchasePrice * 100) : 0`
- `cashLeftIn = totalCostIn - refiAmount`
- `cashOnCashHold = cashLeftIn > 0 ? ((monthlyCashflow * 12) / cashLeftIn * 100) : 0`

**Return object:**
```js
return {
  sdlt, loanAmount, bridgingInterest, arrangementFee, solicitor, survey, brokerFee,
  totalCostIn, cashIn, usingFallback, bestLenderName: bestLender?.name || null,
  flip: { netProfit, roi, cashOnCash },
  hold: rentalIncome > 0 ? { refiAmount, btlMortgage, management, voidAllowance, monthlyCashflow, annualYield, cashLeftIn, cashOnCashHold } : null
};
```
</details>

<title>Implement live recalculation with debounce and results rendering</title>
<details>
Add the following functions in `index.html` near the deal stacking section:

**Debounce helper:**
```js
let _dsTimers = {};
function debounceDealStack(idx) {
  clearTimeout(_dsTimers[idx]);
  _dsTimers[idx] = setTimeout(() => runDealStack(idx), 300);
}
```

**Main render function:**
```js
function runDealStack(idx) {
  const lot = window._currentLots?.[idx];
  if (!lot || !lot.price) return;

  const gdv = parseFloat(document.getElementById('ds-gdv-' + idx)?.value) || 0;
  const works = parseFloat(document.getElementById('ds-works-' + idx)?.value) || 0;
  const rental = parseFloat(document.getElementById('ds-rental-' + idx)?.value) || 0;
  const ltvSlider = document.getElementById('ltv-slider-' + idx);
  const ltv = ltvSlider ? parseInt(ltvSlider.value) : 70;

  if (gdv <= 0) {
    document.getElementById('ds-results-' + idx).innerHTML =
      '<p style="color:var(--text-muted); font-size:13px; text-align:center; padding:12px 0;">Enter a GDV to see your deal stack</p>';
    return;
  }

  const result = calcDealStack(lot.price, gdv, works, rental, ltv, lot.address || '');
  renderDealStackResults(idx, result, rental > 0);
}
```

**Results rendering function `renderDealStackResults(idx, r, showHold)`:**

<task id="03-01-05">
<title>Handle edge cases in deal stacking calculations</title>
<details>
Add defensive checks in `calcDealStack()`:

1. **£0 or missing price (TBA lots):** If `purchasePrice <= 0 || isNaN(purchasePrice)`, return null. In `runDealStack()`, check for null return and show "Guide price not available — deal stacking requires a price" message.

2. **Very high prices (>£1.5M):** No special handling needed — `calcSDLT()` already handles all bands correctly. But verify the top SDLT band applies.

3. **GDV lower than purchase price (loss-making deal):** Allow it — show negative profit in red. This is valid information for the user.

4. **Zero works cost:** Allow it — some lots need no refurb. Works cost = 0 is a valid input.

5. **Hold scenario with cashLeftIn <= 0:** This means the refinance covers all costs — show "No cash left in deal" and cashOnCashHold as "N/A" (infinite return). Use a special case: if `cashLeftIn <= 0`, display "All capital recycled" in green.

6. **LENDER_DATA not yet loaded:** `window.LENDER_DATA` might be undefined on first load before the API call completes. Handle with `(window.LENDER_DATA || [])` which the lender matching logic already does.

7. **Missing address for country detection:** If `address` is empty/undefined, `detectCountry()` defaults to 'england', which is the correct fallback.

Add these checks inline in `calcDealStack()` and `runDealStack()`.
</details>
