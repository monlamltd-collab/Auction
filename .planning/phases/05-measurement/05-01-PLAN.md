**3. Create lightweight tracking endpoint for client-only actions.**
Add BEFORE the activity logging section (before line 11174), inside the admin/API section:
```javascript
// ── Lightweight event tracking for client-only actions ──
app.post('/api/track/event', rateLimit(60000, 30), async (req, res) => {
  const { action, detail } = req.body || {};
  const allowed = ['deal_stacking', 'csv_export', 'bridgematch_open'];
  if (!action || !allowed.includes(action)) return res.status(400).json({ error: 'Invalid action' });
  const user = await validateUserFromReq(req).catch(() => null);
  logActivityEvent(action, detail || {}, user?.email || null, getClientIP(req));
  res.json({ ok: true });
});
```
This endpoint:
- Accepts only the 3 allowed client-only action names (deal_stacking, csv_export, bridgematch_open)
- Rate limited to 30/min per IP
- Extracts user email from JWT if signed in
- Fire-and-forget (does not await logActivityEvent before responding)
- Does NOT require auth (anonymous tracking allowed, consistent with how analysis/smart_search work)

**Important:** The existing 3 events (analysis, smart_search, lead_submit) remain untouched. Include IP on all new events via `getClientIP(req)` per user decision.
