# Plan 01-03: Firecrawl Format, Lot Status Pipeline & Future-Only Display

<objective>
Switch Firecrawl default format to include markdown for better Gemini extraction, add a standardised `lot.status` field to the extraction pipeline, render sold/STC/withdrawn overlays on lot cards, and change the frontend to default to future-only auctions with server-side filtering and a 7-day grace period.
</objective>

<must_haves>
- Firecrawl requests use `['markdown', 'rawHtml']` format by default
- Every lot has a `lot.status` field with value 'available', 'sold', 'stc', or 'withdrawn'
- Sold/STC/withdrawn lots display a diagonal overlay banner on the lot card image
- Frontend defaults to future-only auctions (past auctions hidden unless user opts in)
- Server-side filtering via `?includePast=true` with 7-day grace period
- URL parameter `?showPast=true` is readable and shareable
</must_haves>

<task id="01-03-03" title="Add standardised lot.status field to extraction pipeline">
<action>

   **Find patterns like:** `lot.bullets.push('SOLD/STC')`
   **Replace with:** `lot.status = 'sold'` (or 'stc' depending on detection)

   For all other extractors that don't detect status, ensure `lot.status` defaults to `'available'`.

2. **Post-extraction normalisation:** After lots are extracted (either DOM or AI), add a normalisation step:

   ```javascript
   // Normalise lot.status
   lots.forEach(lot => {
     if (!lot.status) {
       // Check bullets for legacy status detection
       const bulletStr = (lot.bullets || []).join(' ');
       if (/\bSOLD\b/i.test(bulletStr)) lot.status = 'sold';
       else if (/\bSTC\b|\bSALE.?AGREED\b/i.test(bulletStr)) lot.status = 'stc';
       else if (/\bWITHDRAWN\b/i.test(bulletStr)) lot.status = 'withdrawn';
       else lot.status = 'available';
     }
   });
   ```

3. **Gemini extraction prompts:** Find the prompt template used in `extractLotsWithAI()` and add `status` to the expected JSON schema:
   `"status": "one of: available, sold, stc, withdrawn — default available if not stated"`

4. **AI search regex:** Update the AI search endpoint (~line 3311) that uses regex on bullets to also check `lot.status` field directly, preferring the structured field over regex.
</action>

<task id="01-03-06" title="Add server-side future-only filtering to /api/all-lots">
<action>
In `server.js`, find the `/api/all-lots` route handler (~line 1135 per research). Add query parameter support:

```javascript
app.get('/api/all-lots', async (req, res) => {
  const includePast = req.query.includePast === 'true';

  // ... existing lot retrieval logic ...

  if (!includePast) {
    const cutoff = new Date();
    cutoff.setDate(cutoff.getDate() - 7); // 7-day grace period

    lots = lots.filter(lot => {
      // Include lots with no date (don't hide valid lots)
      if (!lot._auctionDate) return true;
      const auctionDate = new Date(lot._auctionDate);
      return auctionDate >= cutoff;
    });
  }

  // ... rest of response logic ...
});
```

Key: lots with `_auctionDate: null` must be included by default (per research risk assessment — hiding them would lose valid lots).
