  <name>Task 1: Create lib/ai-provider.js with provider abstraction, cost logging, and Supabase schema</name>
  <files>lib/ai-provider.js, schema.sql</files>
  <action>
Create the `lib/` directory and `lib/ai-provider.js` module. This is the core provider abstraction layer.

**Provider registry (PROVIDERS object):**
- `gemini`: fast = 'gemini-2.5-flash-lite', capable = 'gemini-2.5-pro'. Uses existing `@google/generative-ai` SDK. Pricing: flash-lite $0.10/$0.40, pro $1.25/$10.00 per 1M tokens.
- `grok`: fast = 'grok-4-1-fast-non-reasoning', capable = 'grok-4-1-fast-reasoning'. Uses native fetch to `https://api.x.ai/v1/chat/completions`. Pricing: $0.20/$0.50 per 1M tokens.

**Main export: `callAI(prompt, opts)`:**
- Options: `{ tier = 'fast', maxTokens = 8000, systemPrompt = null, pdfBase64 = null, taskType = 'extraction' }`
- Reads `AI_PROVIDER` env var (default 'gemini') to select provider
- CRITICAL: When `pdfBase64` is provided, ALWAYS use Gemini regardless of AI_PROVIDER (Grok doesn't support multimodal)
- Each provider's call function returns `{ text, usage: { promptTokenCount, candidatesTokenCount } }`
- For Gemini: extract `response.usageMetadata` (currently discarded by callGemini). Use the existing `@google/generative-ai` SDK import and `genAI` instance -- these must be passed in or re-created. Since server.js already creates the genAI instance, export an `initAI(genAIInstance, supabaseClient)` function that the server calls at startup to inject dependencies.
- For Grok: parse `data.usage.prompt_tokens` / `data.usage.completion_tokens` from OpenAI-format response

**Rate limiting:**
- Move `geminiRateLimited` logic into the Gemini provider call function
- Add separate rate limiter for Grok (configurable via `GROK_MIN_GAP_MS` env var, default 100ms)

**Cost logging (`logAICost()`):**
- Fire-and-forget async insert to Supabase `ai_usage` table (never await in critical path)
- Fields: provider, model, tokens_in, tokens_out, est_cost, task_type, duration_ms, created_at
- `estimateCost()` uses the pricing lookup table in the provider registry
- Follows the same pattern as `logActivityEvent()` -- try/catch with console.warn on failure

**Daily budget enforcement:**
- In-memory `dailyCostTotal` counter incremented after each call (avoids DB query race condition per Pitfall 6 in RESEARCH.md)
- Resets at midnight via `setInterval`
- Before each callAI(), check `dailyCostTotal >= AI_DAILY_BUDGET`. If exceeded, log warning and still proceed (soft cap) but set a flag `budgetExceeded = true` that the admin endpoint can read
- Export `getAICostSummary()` for the admin endpoint to read in-memory stats

**Second export: `getAICostSummary()`:**
- Returns `{ dailyCostTotal, budgetExceeded, callCount, budget }` from in-memory state

**Append to schema.sql** the ai_usage table DDL:
```sql
CREATE TABLE IF NOT EXISTS ai_usage (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  provider TEXT NOT NULL,
  model TEXT NOT NULL,
  tokens_in INTEGER DEFAULT 0,
  tokens_out INTEGER DEFAULT 0,
  est_cost NUMERIC(10,6) DEFAULT 0,
  task_type TEXT,
  duration_ms INTEGER,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_ai_usage_created ON ai_usage(created_at DESC);
CREATE INDEX IF NOT EXISTS idx_ai_usage_provider ON ai_usage(provider, created_at DESC);
ALTER TABLE ai_usage ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Service role full access" ON ai_usage FOR ALL USING (true) WITH CHECK (true);
```

**Environment variables** (add to .env.example if it exists):
- `AI_PROVIDER` (default: 'gemini')
- `GROK_API_KEY` (only needed when AI_PROVIDER=grok)
- `AI_DAILY_BUDGET` (default: '0.50', in USD)
- `GROK_MIN_GAP_MS` (default: '100')
