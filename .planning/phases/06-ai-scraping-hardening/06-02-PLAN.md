  <action>
**Add new audit phase: LOT-COUNT CROSS-VALIDATION** (insert after existing Phase 3 or 4).

For each house that has a successful Puppeteer/HTTP probe result:

1. **Extract house-advertised lot count** from the probe's page content. Use heuristic regex patterns on the page text:
   - Pattern 1: `(\d+)\s+(?:lots?|properties|results)` (e.g., "45 Lots")
   - Pattern 2: `of\s+(\d+)` (e.g., "Showing 1-20 of 45")
   - Pattern 3: `(\d+)\s+(?:upcoming|available|current)`
   - If no match found, mark as "stated count unavailable" (not a failure)

2. **Compare to production lot count** from the existing Phase 4 production comparison data (which already fetches `/api/all-lots`). For each house, extract the cached lot count from the production API response.

3. **Flag mismatches** when stated count > extracted count * 1.3 (30% tolerance to account for pagination/timing):
   ```
   { type: 'LOT_COUNT_MISMATCH', severity: 'WARNING', house, statedCount, extractedCount, ratio }
   ```
   If stated count > extracted count * 2.0, escalate to severity 'BROKEN'.

4. **Output format**: Add a "Lot Count Validation" section to the terminal report showing:
   ```
   Lot Count Validation:
     savills:       stated 45, extracted 42  [OK]
     allsop:        stated 30, extracted 12  [MISMATCH - 40% coverage]
     bidx1:         stated count unavailable [SKIP]
   ```

**Enhance image coverage reporting:**

In the existing production comparison phase, add per-house image coverage calculation:
- For each house's lots from production API, count lots with non-empty `imageUrl`
- Calculate `imageCoverage = lotsWithImages / totalLots * 100`
- Flag houses below 90%: `{ type: 'LOW_IMAGE_COVERAGE', house, coverage, lotsWithImages, totalLots }`
- Add "Image Coverage" section to terminal report:
  ```
  Image Coverage:
    savills:       42/45 (93%)  [OK]
    allsop:        8/30  (27%)  [BELOW TARGET]
  ```

**Add --validate flag** to run just lot-count validation + image coverage (faster than full audit for routine checks).

**JSON output**: When `--json` flag is used, include `lotCountValidation` and `imageCoverage` objects in the output.

3. **Modify `extractWithJSDOM()`** (line ~4660): Before running the DOM extractor, check if the house is in BROKEN_EXTRACTORS. 5. **Load broken extractors from Supabase on startup.** Add a `house_health` or use existing `house_skills` table to persist broken status. On startup, load broken houses into the BROKEN_EXTRACTORS set. When the disable/enable endpoints are called, persist to Supabase as well.
