**Part A: Add GET /api/admin/system-health endpoint to server.js.**

This is a consolidated endpoint that gathers health data from multiple sources into one response. Admin-authenticated.

Response shape:
```json
{
  "brokenExtractors": ["house1", "house2"],
  "aiCosts": { "dailyTotal": 0.12, "budget": 0.50, "overBudget": false, "byModel": {...}, "callCount": 15 },
  "coverage": {
    "houses": [
      { "slug": "savills", "displayName": "Savills", "lotCount": 45, "imageCoverage": 93, "status": "active", "lastScraped": "2026-03-22T10:00:00Z" },
      ...
    ],
    "totalHouses": 25,
    "activeHouses": 21,
    "staleHouses": 4,
    "totalLots": 2364,
    "avgImageCoverage": 87
  },
  "pipeline": {
    "firecrawl": { "status": "ok", "creditsUsed": 1200, "creditBudget": 15000, "exhausted": false },
    "gemini": { "status": "ok", "exhausted": false, "provider": "gemini" },
    "puppeteer": { "status": "available", "available": true },
    "autoAnalyse": { "running": false, "lastRun": "2026-03-22T06:00:00Z", "nextRun": "2026-03-22T12:00:00Z" }
  }
}
```

Build this by reading from:
- BROKEN_EXTRACTORS set (from Plan 02)
- getAICostSummary() + Supabase ai_usage query (from Plan 01)
- Supabase cached_analyses for per-house lot counts and image coverage
- Supabase house_skills for house status and last scraped time
- In-memory flags: fcCreditExhausted, creditExhausted, puppeteer availability
- autoAnalyseAll running state
