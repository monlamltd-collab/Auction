# Plan 01-04: Admin Alerting & Data Freshness Metrics

<objective>
Add a pipeline alerting system that detects auto-analyse failures, discovery misses, image coverage drops, and DOM extractor regressions, displaying them in a new "Alerts" feed on the admin dashboard. Add a per-house health table with last scrape time, lot count, image coverage %, status, and diff summaries per scrape run, giving the admin a clear view of data freshness and pipeline health.
</objective>

<must_haves>
- Pipeline failures (auto-analyse, discovery) generate alert records within the same run
- Alerts are visible in admin.html in a chronological feed with severity indicators
- Alerts auto-resolve when the next successful run clears the issue
- Per-house health table shows: house name, last scrape time, lot count, image coverage %, health status
- Each scrape run records a diff summary: lots added/removed/changed, images gained/lost
- Admin dashboard data is protected by `x-admin-secret` header (existing auth pattern)
</must_haves>

Add the following table to `schema.sql` and execute as a migration:

```sql
CREATE TABLE IF NOT EXISTS pipeline_alerts (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  event_type TEXT NOT NULL,          -- 'auto_analyse_failure', 'discovery_miss', 'image_coverage_drop', 'extractor_regression'
  severity TEXT NOT NULL DEFAULT 'warning',  -- 'warning' or 'error'
  house TEXT,                        -- auction house slug (null for system-wide alerts)
  message TEXT NOT NULL,
  resolved BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  resolved_at TIMESTAMPTZ
);

CREATE INDEX idx_pipeline_alerts_resolved ON pipeline_alerts(resolved, created_at DESC);
```

Event types and severity mapping:
- `auto_analyse_failure` + error: autoAnalyseOne threw an exception or returned 0 lots
- `discovery_miss` + warning: discoverAndUpdateCalendar found no new catalogues
- `image_coverage_drop` + warning: house image coverage dropped below 50%
- `extractor_regression` + error: DOM extractor returns 0 lots when it previously returned >0
</action>

<task id="01-04-06" title="Compute and store per-scrape diff summaries">
<action>
In `server.js`, in the `autoAnalyseOne()` function, after lots are extracted and before caching:

1. Load the previous lot set for this house from cache or `cached_analyses`
2. Compute the diff:
   ```javascript
   function computeScrpeDiff(oldLots, newLots) {
     const oldMap = new Map((oldLots || []).map(l => [l.lotNumber || l.address, l]));
     const newMap = new Map(newLots.map(l => [l.lotNumber || l.address, l]));
     const added = [...newMap.keys()].filter(k => !oldMap.has(k));
     const removed = [...oldMap.keys()].filter(k => !newMap.has(k));
     const changed = [...newMap.keys()].filter(k => {
       if (!oldMap.has(k)) return false;
       const o = oldMap.get(k), n = newMap.get(k);
       return o.price !== n.price || o.status !== n.status;
     });
     const imagesGained = newLots.filter(l => l.imageUrl && !(oldMap.get(l.lotNumber || l.address)?.imageUrl)).length;
     const imagesLost = (oldLots || []).filter(l => l.imageUrl && !(newMap.get(l.lotNumber || l.address)?.imageUrl)).length;

     const summary = [];
     if (added.length) summary.push(`+${added.length} new lots`);
     if (removed.length) summary.push(`${removed.length} removed`);
     if (changed.length) summary.push(`${changed.length} changed`);
     if (imagesGained) summary.push(`${imagesGained} images added`);
     if (imagesLost) summary.push(`${imagesLost} images lost`);

     return {
       lots_added: added.length,
       lots_removed: removed.length,
       lots_changed: changed.length,
       images_gained: imagesGained,
       images_lost: imagesLost,
       status_changes: summary,
       timestamp: new Date().toISOString()
     };
   }
   ```

3. Store the diff in `house_skills.last_diff`:
   ```javascript
   await supabase.from('house_skills')
     .update({ last_diff: diff })
     .eq('slug', houseSlug);
   ```

```javascript
app.get('/api/admin/alerts', async (req, res) => {
  if (!safeCompare(req.headers['x-admin-secret'], ADMIN_SECRET)) {
    return res.status(401).json({ error: 'Unauthorized' });
  }
  try {
    // Active alerts first, then recently resolved
    const { data: active } = await supabase
      .from('pipeline_alerts')
      .select('*')
      .eq('resolved', false)
      .order('created_at', { ascending: false })
      .limit(50);

    const { data: recent } = await supabase
      .from('pipeline_alerts')
      .select('*')
      .eq('resolved', true)
      .order('resolved_at', { ascending: false })
      .limit(20);

    res.json({ active: active || [], recent: recent || [] });
  } catch (err) {
    log.error('Failed to fetch alerts:', err);
    res.status(500).json({ error: 'Failed to fetch alerts' });
  }
});
```

```javascript
async function loadFreshness() {
  const resp = await fetch('/api/admin/freshness', {
    headers: { 'x-admin-secret': adminSecret }
  });
  const { houses } = await resp.json();

  document.getElementById('freshnessBody').innerHTML = houses.map(h => {
    const statusClass = h.status === 'healthy' ? 'status-healthy' :
                        h.status === 'degraded' ? 'status-warning' : 'status-broken';
    const statusLabel = h.status === 'healthy' ? 'Healthy' :
                        h.status === 'degraded' ? 'Warning' : 'Broken';
    const lastScrape = h.last_verified ? new Date(h.last_verified).toLocaleString() : 'Never';
    const diffSummary = h.last_diff?.status_changes?.join(', ') || 'No changes';

    return `<tr>
      <td>${escHtml(h.name || h.slug)}</td>
      <td>${lastScrape}</td>
      <td>${h.lot_count || 0}</td>
      <td>${(h.image_coverage || 0).toFixed(0)}%</td>
      <td><span class="status-badge ${statusClass}">${statusLabel}</span></td>
      <td><small>${escHtml(diffSummary)}</small></td>
    </tr>`;
  }).join('');
}
```
