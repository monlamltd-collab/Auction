# Plan 02: Image Coverage Improvement & Missing Image Admin Tooling

## Goal
Raise image coverage above 80% across all auction houses by improving extraction strategies and Firecrawl structured output usage, and add an admin dashboard feature to flag and filter lots with missing images.

## must_haves
- [ ] Image coverage exceeds 80% across all houses (measurable via admin dashboard metric)
- [ ] Firecrawl structured output (`images` format) leveraged for better image extraction
- [ ] Missing-image lots visible in admin dashboard with filterable list (house, lot number, catalogue URL)

## Tasks

<task id="02-02-01">
<title>Audit and fix DOM extractor image selectors</title>
<details>
Review the `DOM_EXTRACTORS` object (starting at line 5149 in `server.js`) and fix common image extraction issues across all extractors.

**Systematic fixes to apply per-extractor:**
1. Check for `data-src`, `data-lazy-src`, `data-original` attributes in addition to `src` — many sites use lazy-loading. Add fallback chain: `img.getAttribute('data-src') || img.getAttribute('data-lazy-src') || img.getAttribute('data-original') || img.src`
2. Check for background images in `style` attributes: `el.style.backgroundImage.match(/url\(['"]?(.+?)['"]?\)/)`
3. Replace thumbnail URL patterns with full-size equivalents: common patterns include `/thumb/` → `/large/`, `/small/` → `/medium/`, `_thumb.` → `.`, `w=100` → `w=800`
4. Resolve relative URLs: if imageUrl starts with `/`, prepend the house's base URL from `HOUSE_ROOTS`
5. Filter out icon/logo/nav images: skip images with dimensions < 50px, or with paths containing `logo`, `icon`, `nav`, `sprite`, `placeholder`

**Priority houses:** Focus on houses with lowest `house_skills.image_coverage` values first. Check the `HOUSE_ROOTS` entries that have extractors but historically low image rates.

**Do not break working extractors** — only add fallback logic, not replace existing working selectors.
</details>

<task id="02-02-02">
<title>Enhance Firecrawl image extraction with structured output</title>
<details>
Modify `scrapeRenderedPage()` and `extractWithJSDOM()` in `server.js` to better leverage Firecrawl's image capabilities.

**Changes to `scrapeRenderedPage()`:**
1. Ensure the Firecrawl request format includes `'images'` alongside `['markdown', 'rawHtml']` — making it `['markdown', 'rawHtml', 'images']` if not already (verify current format array)
2. The `images` response contains an array of all page image URLs — pass this to `extractWithJSDOM()` as the `firecrawlImages` parameter

**Changes to `extractWithJSDOM()` and image matching:**
1. In the image matching logic, after DOM extractor runs, for any lot still missing `imageUrl`:
   - Filter `firecrawlImages` to remove obvious non-lot images (logos, icons, sprites — check for dimension hints or path patterns)
   - Attempt matching by lot URL path overlap (lot URL substring appears in image URL)
   - Attempt matching by lot number in image filename
   - Last resort: position-based matching with improved accuracy — count only "content" images (skip header/footer/nav images by filtering URLs containing `/assets/`, `/static/`, `/icons/`)
2. Add logging: `console.log('[IMG] ${house}: ${lotsWithImages}/${totalLots} lots have images after Firecrawl merge')`

**Changes to `backfillImagesFromLotPages()`:**
1. Ensure this function runs for ALL houses, not just a hardcoded subset
2. For each lot missing an image, fetch the lot's detail page URL and extract the first content image (skip nav/logo/placeholder images)
3. Respect rate limits: 500ms delay between lot page fetches, max 50 lot pages per catalogue run
</details>

<task id="02-02-03">
<title>Add missing-image lots admin API endpoint</title>
<details>
Add a new admin endpoint in `server.js` for querying lots with missing images.

**Endpoint:** `GET /api/admin/missing-images`

**Query params:**
- `house` (optional) — filter by auction house slug
- `limit` (optional, default 100) — max results
- `offset` (optional, default 0) — pagination

**Implementation:**
1. Query `cached_analyses` table for all rows where `auction_date >= NOW()` (future auctions only)
2. For each row, iterate the `lots` JSONB array and filter for lots where `imageUrl` is null, empty string, or undefined
3. Return JSON array: `[{ house, lotNumber, address, catalogueUrl, auctionDate }]`
4. Protect with `x-admin-secret` auth (same pattern as existing admin endpoints like `POST /api/admin/calendar`)

**Performance:** Use Supabase JSONB operators to filter server-side where possible. If JSONB querying is too complex, fetch and filter in Node.js but limit to recent/future catalogues only.
</details>
