Add as many new UK auction houses as possible (minimum 5) with working DOM extractors that return lots on live test, capture images where available, and handle pagination correctly.

**For each new custom extractor:**

2. Inspect the HTML structure to identify lot card selectors
3. Write the DOM extractor JS string following the existing pattern:
   - `document.querySelectorAll('{lot-card-selector}')` to find lot elements

   - Return array of lot objects
4. Add to `DOM_EXTRACTORS` object with the house slug as key
5. Add to `HOUSE_ROOTS` with catalogue URL
6. Add to `detectAuctionHouse()` URL pattern matching if needed

**If a house consistently returns 403 even via Firecrawl**, do not add it — add a comment in the code noting it blocks scraping. Focus effort on houses that work.

<task id="02-03-04">
<title>Implement and verify pagination for new multi-page houses</title>
<details>
For each new house added in tasks 02-03-01 through 02-03-03, verify and implement pagination handling.

**Check each new house for pagination:**
1. Load the catalogue page and check for pagination indicators:
   - Page number links (`?page=2`, `?p=2`, `/page/2/`)
   - "Next" button
   - Infinite scroll / AJAX loading
   - Total lot count vs visible lot count mismatch
2. If paginated, implement the pagination pattern in the house's scraping flow

**Pagination patterns to handle (match existing patterns in `scrapePageWithFirecrawl`):**
- **URL parameter:** Append `?page=N` or `&page=N` — loop until page returns 0 new lots
- **Path-based:** Append `/page/N/` to URL
- **API cursor:** Some sites use AJAX endpoints with cursor/offset params — intercept the API call pattern
- **Cap:** Respect `MAX_PUPPETEER_PAGES` limit (check existing value in server.js) to prevent runaway pagination

**Verification per paginated house:**
- Page 1 lots + Page 2 lots > Page 1 lots alone
- No duplicate lot numbers across pages
- All pages have same extraction quality (addresses, prices, images)

**Log pagination:** For each house, log `[PAGINATION] ${house}: ${totalPages} pages, ${totalLots} lots`
</details>
