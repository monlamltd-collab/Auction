# Plan 01: EPC & Flood Risk Enrichment Pipeline

## Goal
Add EPC rating and flood risk zone enrichment to every lot with a valid postcode, using MHCLG open data and Environment Agency APIs, with 30-day postcode-level caching, available to all users regardless of tier.

## must_haves
- [ ] Lots with valid postcodes display EPC rating (A-G) on the lot detail card
- [ ] Lots with valid postcodes display flood zone (1/2/3) on the lot detail card
- [ ] Enrichment results cached at postcode level with 30-day TTL in `enrichment_cache` table
- [ ] Enrichment data visible to all users (not gated behind premium tier)

**Error handling:** On 4xx/5xx, log warning and return null (enrichment is best-effort, never blocks pipeline).

<task id="02-01-03">
<title>Implement EPC address matching logic</title>
<details>
Add `matchEPCToLot(epcRecords, lotAddress)` function in `server.js`.

**Implementation:**
1. Normalise both EPC address and lot address: lowercase, strip "flat", "apartment", "unit", collapse whitespace, remove commas
2. For each EPC record, compute a similarity score against the lot address (substring match on house number + street name)
3. If multiple matches, pick the one with the most recent `lodgement-date`
4. Return `{ epcRating: record['current-energy-rating'], epcScore: record['current-energy-efficiency'], epcDate: record['lodgement-date'] }` or null if no confident match

**Confidence threshold:** Only match if the normalised lot address contains the EPC record's building number AND street name. Return null rather than a wrong match.

**Validation:** `epcRating` must be a single character A-G, `epcScore` must be 1-100.
</details>

<task id="02-01-04">
<title>Implement flood zone lookup via Postcodes.io + EA WFS</title>
<details>
Add `fetchFloodZone(postcode)` function in `server.js`.

**Implementation (two-step):**
1. **Geocode:** Call `https://api.postcodes.io/postcodes/{postcode}` to get lat/lon. Cache lat/lon in `enrichment_cache.lat`/`lon`. Use Postcodes.io bulk endpoint (`POST /postcodes` with array of up to 100) when processing multiple postcodes.
2. **Flood zone query:** Call EA WFS endpoint:
   - Flood Zone 3: `https://environment.data.gov.uk/spatialdata/flood-map-for-planning-rivers-and-sea-flood-zone-3/wfs?service=WFS&version=2.0.0&request=GetFeature&typeNames=Flood_Map_for_Planning_Rivers_and_Sea_Flood_Zone_3&outputFormat=application/json&count=1&srsName=EPSG:4326&CQL_FILTER=INTERSECTS(shape,POINT({lon} {lat}))`
   - If Zone 3 hit: `floodZone = "3"`, `floodRiskLevel = "High"`
   - If no Zone 3 hit, check Zone 2 with equivalent URL
   - If Zone 2 hit: `floodZone = "2"`, `floodRiskLevel = "Medium"`
   - If neither: `floodZone = "1"`, `floodRiskLevel = "Low"`

**Fallback:** If WFS returns errors or times out (5s timeout), try the simpler flood monitoring API: `https://environment.data.gov.uk/flood-monitoring/id/floods?lat={lat}&long={lon}&dist=5`. If any active warnings: `floodRiskLevel = "Alert"`. If no warnings: `floodRiskLevel = "Low"`.

**If WFS is unreliable across multiple postcodes**, disable WFS and use flood monitoring API only. Log which approach is used.

**Rate limiting:** 200ms delay between EA API calls, CONCURRENCY=3.
</details>

<task id="02-01-05">
<title>Integrate enrichment into enrichLots() pipeline with caching</title>
<details>
Modify the existing `enrichLots()` function in `server.js` to add EPC + flood enrichment after the existing Land Registry enrichment step.

**Implementation:**
1. After existing Land Registry queries (which already group lots by postcode), add a new loop over unique postcodes
2. For each postcode:
   a. Check `enrichment_cache`: `SELECT * FROM enrichment_cache WHERE postcode = $1 AND expires_at > NOW()`
   b. If cache hit: use stored `epc_data` and `flood_zone`
   c. If cache miss:
      - Call `fetchEPCByPostcode(postcode)` → get EPC records
      - Call `fetchFloodZone(postcode)` → get flood zone + lat/lon
      - Insert into `enrichment_cache`: postcode, epc_data (full array), flood_zone, flood_data, lat, lon
   d. For each lot with this postcode:
      - Call `matchEPCToLot(epcRecords, lot.address)` → set `lot.epcRating`, `lot.epcScore`, `lot.epcDate`
      - Set `lot.floodZone` and `lot.floodRiskLevel` from cache
      - Set `lot.enrichedAt` = new Date().toISOString()
3. Use CONCURRENCY=3 with `Promise.allSettled` for parallel postcode processing (same pattern as existing Land Registry batching)
4. Periodically clean expired cache entries: `DELETE FROM enrichment_cache WHERE expires_at < NOW()` — run this once per `autoAnalyseAll()` cycle

**Graceful degradation:** If EPC or flood APIs are down, lots still get the other enrichment data. If both fail, lots proceed without enrichment (null fields).
</details>
